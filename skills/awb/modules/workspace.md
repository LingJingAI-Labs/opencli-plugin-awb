# Workspace Module

账号、团队、项目组决定权限、积分和任务归属。创作前不要假设当前项目组是对的。

## 1. 层级

- 团队：上层组织，用 `groupId` 标识。
- 项目组：真正的计费、权限、作品和任务归属单位，用 `projectGroupNo` 标识。
- 当前团队 / 当前项目组会写入本地配置，后续命令默认使用它们。
- `me` 是最省的总览：账号、权限、当前团队、当前项目组、积分和登录有效期。

## 2. 命令入口

| 命令 | 用途 |
|------|------|
| `me` | 快速 sanity check |
| `teams` / `team-select` | 查看和切换团队 |
| `project-groups` / `project-group-select` | 查看和切换项目组 |
| `project-group-current` | 查看当前项目组详情 |
| `project-group-create` / `project-group-update` | 创建或更新项目组 |
| `project-group-users` | 查可加入项目组的团队成员 |

具体参数以 `--help` 为准；写入类命令不确定时先 dry-run。

## 3. 经验

- 创作前要主动告诉用户：当前团队、当前项目组、可用项目组列表，以及本次任务预计会记到哪个项目组。
- 用户提到客户、项目、预算、交付批次或多人协作时，先判断当前项目组是否匹配；不匹配就问是否切换，或本次命令临时加 `--projectGroupNo`。
- 创作失败时先查当前项目组和 `points`，不是只看团队余额。
- 临时跑到另一个项目组时，优先传 `--projectGroupNo` 覆盖，不一定要切换全局当前项目组。
- 切团队后必须重新核对当前项目组；CLI 可能自动选最近使用项。
- 新建项目组默认额度语义以 CLI 当前 help 为准；需要限额时显式设置。
- `--userIds` 只能从 `project-group-users` 的结果里选。
- 多 agent / 多账号共机时，每次正式提交前把项目组写进确认摘要。
