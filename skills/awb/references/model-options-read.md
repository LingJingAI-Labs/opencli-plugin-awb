# 读懂 `model-options`

`model-options` 是实时真相；不要把 skill 里的历史经验当参数白名单。

## 必看点

- 看 `推荐CLI用法`，不要把 `底层参数` 直接当命令行 flag。
- 看 `约束`：枚举值、默认值、文件数上限、文件格式、prompt 长度。
- 参数可能有条件性约束；如果 CLI 提示需要已选参数，再用 `--selectedConfigsJson` 复查。
- `-f json` 输出只供 agent 管道解析；长输出先用 `jq` / `node` 摘要。

## 经验

- 每次换模型或模型组都重查。即便同一家族，不同通道的参数也可能不一样。
- 有默认值不代表可以忽略用户选择；清晰度、比例、时长、候选图数量这类关键项仍要确认。
- 参考类型对用户说自然语言：参考图片、角色参考图片、风格参考图片；底层 `iref` / `cref` / `sref` 只在写命令时使用。
- 看到文件数上限时要把所有参考素材都算进去；分镜指挥图本身也算一张参考图片。
